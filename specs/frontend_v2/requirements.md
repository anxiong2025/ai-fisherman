# 需求文档

## 介绍

本项目旨在对现有的静态博客网站进行升级，引入动态功能，包括用户登录、后台管理、Markdown 文章支持、支付预留以及 AI 聊天界面。目标是实现一个功能更完善的内容管理系统（CMS）和用户互动平台。

## 需求

### 需求 1 - 用户登录

**用户故事：** 作为用户，我希望能通过 GitHub 或 Google 账号快速登录网站，以便进行评论或访问特定内容。

#### 验收标准
1.  **登录入口**：导航栏提供"登录"按钮。
2.  **登录方式**：支持 GitHub 和 Google OAuth 快捷登录。
3.  **技术实现**：使用 CloudBase Web SDK 的 `auth` 模块。
4.  **状态保持**：登录后保持会话状态，刷新页面不丢失登录态。
5.  **用户信息**：登录后显示用户头像和昵称。

### 需求 2 - 后台管理系统

**用户故事：** 作为管理员，我希望能有一个后台界面来发布文章和管理课程，而不需要手动修改 HTML 代码。

#### 验收标准
1.  **访问权限**：仅管理员可访问（需定义管理员角色或特定账号）。
2.  **文章管理**：
    *   支持创建新文章。
    *   字段包括：标题、分类、标签、封面图、内容。
    *   内容编辑器支持 Markdown 粘贴和预览。
3.  **课程管理**：
    *   支持课程的上架/下架操作。
    *   支持编辑课程基本信息（标题、描述、价格）。

### 需求 3 - 文章 Markdown 支持

**用户故事：** 作为读者，我希望能看到排版优美的文章；作为作者，我希望能直接复制 Markdown 内容进行发布。

#### 验收标准
1.  **渲染引擎**：前端集成 Markdown 解析库（如 `marked` 或 `markdown-it`）。
2.  **样式适配**：Markdown 渲染后的 HTML 需适配当前网站的 CSS 风格。
3.  **代码高亮**：支持代码块语法高亮。

### 需求 4 - 支付功能（预留）

**用户故事：** 作为管理员，我希望为未来的课程销售预留支付接口。

#### 验收标准
1.  **支付按钮**：课程详情页显示"购买"按钮。
2.  **对接准备**：点击购买按钮时，模拟调用微信支付流程（暂不进行实际扣款，仅做流程打通或 UI 展示）。
3.  **扩展性**：代码结构需便于后续接入真实的微信支付 SDK。

### 需求 5 - AI 聊天助手（UI）

**用户故事：** 作为用户，我希望能有一个 AI 聊天窗口，未来用于网站内容检索。

#### 验收标准
1.  **聊天入口**：页面右下角悬浮聊天图标。
2.  **聊天界面**：点击图标展开聊天窗口，包含消息列表和输入框。
3.  **功能限制**：当前仅实现 UI 和简单的回声（Echo）或预设回复，暂不接入真实 LLM，但需预留接口位置。
4.  **数据范围**：明确标识该助手仅用于"站内检索"（未来功能）。
